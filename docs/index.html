Introduction

The ACG Material Importer extension provides a streamlined workflow for importing materials from AmbientCG.com directly into Blender. It automatically extracts USDZ material archives, imports the USD materials, creates preview spheres, and organizes all texture maps in your shader editor.

Author: Johnny Matthews
Version: 1.1.0
Blender Version: 4.5.0 or later
Category: Material, Import-Export

Installation

This extension can be installed directly from Blender's Extensions repository:

  1. Open Blender 4.5 or later
  2. Go to Edit → Preferences → Get Extensions
  3. Search for "ACG Material Importer" or "AmbientCG"
  4. Click Install

Usage

Prerequisites

Before using the ACG Material Importer:

  • Download material ZIP files from AmbientCG.com (specifically the USDZ format)
  • Know the location of your downloaded ZIP files
  • Ensure you have a valid Blender 4.5+ installation with USD support

Accessing the Importer

The importer can be accessed from:

  • File → Import → AmbientCG USD Materials (.zip)

Basic Workflow

  1. Go to File → Import → AmbientCG USD Materials (.zip)
  2. Navigate to your downloaded AmbientCG ZIP file(s)
  3. Select one or multiple ZIP files
  4. Click Import
  5. The extension will automatically:
    • Extract the ZIP archive
    • Import the USDC material file
    • Add all texture image nodes to the shader
    • Create a preview sphere with the material applied
    • Mark the material as an asset with proper metadata
Note: You can select multiple ZIP files at once to batch import several materials in a single operation.

Features

Automatic Material Import

The extension extracts and imports USDC material files from AmbientCG ZIP archives, preserving all material properties and shader connections.

Texture Node Organization

All texture images found in the ZIP archive are automatically:

  • Loaded into Blender's image library
  • Added as Image Texture nodes in the material's shader editor
  • Arranged in a clean grid layout for easy access
  • Positioned to the right of the Material Output node
  • Checked for duplicates to avoid redundancy

Preview Sphere Creation

Each imported material automatically gets a preview sphere:

  • UV Sphere with 24 segments and rings for smooth appearance
  • Auto-smooth shading applied
  • Material assigned and ready for preview
  • Named after the imported material for easy identification

Asset Management

Materials are automatically tagged as assets with proper metadata:

  • Material names cleaned (removes "aCG_" prefix)
  • Author set to "AmbientCG.com"
  • License set to "Creative Commons CC0 1.0 Universal License"
  • Preview thumbnails automatically generated
  • Ready to use in Blender's Asset Browser

Displacement Support

Materials are configured with displacement method set to 'BOTH' (Displacement + Bump) for optimal rendering.

Batch Import

Select and import multiple ZIP files at once to quickly build your material library.

Tips & Troubleshooting

Finding Materials After Import

After importing, you can find your materials in several places:

  • Scene: Look for the preview sphere in your viewport
  • Shader Editor: The material will be active on the selected sphere
  • Asset Browser: All imported materials are marked as assets
  • Material List: Check the material dropdown in the properties panel

Downloading from AmbientCG

When downloading materials from AmbientCG.com:

  • Make sure to select the USDZ format option
  • Download the ZIP file (not individual textures)
  • No need to extract the ZIP manually - the extension handles this

Working with Texture Nodes

After import, texture nodes are arranged in a grid but not connected to the shader:

  • The imported USD material already has the necessary textures connected
  • Additional texture nodes are provided for easy access to maps like AO, Roughness, etc.
  • Connect these nodes to shader sockets as needed for your workflow

UV Mapping

The extension ensures all UV Map nodes use the default "UVMap" name for consistency with standard Blender objects.

Common Issues

"No .usdc file found in ZIP"

  • Verify you downloaded the USDZ format from AmbientCG
  • Some material formats may not be supported
  • Try re-downloading the file from AmbientCG.com

"No new materials imported"

  • Check if the material already exists in your file
  • Look in the Shader Editor or Material Properties
  • Try importing to a fresh Blender file to test

Textures not showing

  • Ensure you're in Material Preview or Rendered viewport shading
  • Check that texture nodes have images loaded (visible in the node)
  • Verify the preview sphere has the material assigned

Performance Tips

  • Importing many materials at once may take time - be patient
  • Use File → Pack All to embed textures if moving the .blend file
  • Delete preview spheres after import if you don't need them
  • Materials remain in your library even if spheres are deleted
